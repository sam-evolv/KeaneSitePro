Objective

Fix desktop hero overlay to geometric center.

Do not change mobile (≤ 768 px) which is now correct.

Avoid regressions from any prior centering CSS/JS.

Steps

Create /css/hero-center.desktop.css and import it last (after all other CSS).

Do not remove your working mobile centering. We’ll override only for desktop.

/css/hero-center.desktop.css
/* =========================
   DESKTOP-ONLY CENTER LOCK
   ========================= */
@media (min-width: 769px) {
  /* Ensure hero is a positioning context */
  .hero, .hero__media, .hero-video {
    position: relative;
  }

  /* Choose the actual overlay node that holds logo + headings + CTAs.
     We deliberately use higher specificity + !important to beat earlier rules. */
  .hero #hero-centroid-lock,
  .hero .hero__overlay,
  .hero .overlay,
  .hero .hero-content {
    position: absolute !important;
    left: 50% !important;
    top: 50% !important;

    /* Translate to literal geometric center.
       Optional fine-tune via --desktop-center-nudge if needed. */
    transform: translate(-50%, calc(-50% + var(--desktop-center-nudge, 0px))) !important;

    /* Keep group nicely composed but do NOT affect center point */
    display: flex !important;
    flex-direction: column !important;
    align-items: center !important;
    justify-content: center !important;
    text-align: center !important;

    /* Neutralize legacy offsets that push the block off-center */
    margin: 0 !important;
    padding: 0 !important;
    inset: auto !important;
    right: auto !important;
    bottom: auto !important;
    max-width: 1100px;
    width: auto;
    z-index: 3;
    pointer-events: none; /* buttons re-enable below */
  }

  /* Re-enable interactivity for inner elements (buttons/links) */
  .hero #hero-centroid-lock > *,
  .hero .hero__overlay > *,
  .hero .overlay > *,
  .hero .hero-content > * {
    pointer-events: auto;
    margin: 0 auto;
  }

  /* Defensive: strip any transforms from inner children that can offset centering */
  .hero #hero-centroid-lock *,
  .hero .hero__overlay *,
  .hero .overlay *,
  .hero .hero-content * {
    transform: none !important;
  }
}

/* =========================
   MOBILE SAFEGUARD
   (Leave existing mobile centering rules alone)
   ========================= */
@media (max-width: 768px) {
  /* No changes here on purpose */
}

Optional fine-tune (if your header makes the center look a touch high/low)

If you want to visually bias the center slightly on desktop (e.g., –8 px), set once in your global CSS:

:root { --desktop-center-nudge: 0px; } /* try -8px or +8px if you want a tiny visual shift */

Why this fixes it

We use desktop-only media query with higher specificity and !important to beat all earlier mobile/legacy rules.

We center the exact overlay node with left/top:50% + translate(-50%, -50%) — pure geometry.

We neutralize stray margins/transforms that were pulling the block off center.

Mobile rules remain untouched; desktop finally gets a clean, isolated centering model.

Run it, refresh desktop, and the hero logo/text block will sit precisely in the center without disturbing your now-correct mobile layout.
