/* 1) Normalize page + kill stray borders/margins */
html, body {
  height: 100%;
  margin: 0;
  padding: 0;
  background: #000;              /* so any micro-gap blends with video */
  border: 0;
  overflow-x: hidden;
}
* { box-sizing: border-box; }

/* 2) Hero must truly fill the viewport (dynamic units) */
.hero {
  position: relative;
  width: 100%;
  min-height: 100dvh;            /* use dynamic viewport height */
  /* Fallback for older browsers */
  min-height: 100vh;
  overflow: hidden;
  display: grid;
  place-items: center;
}

/* 3) Make the video overscan to hide sub-pixel seams on some GPUs */
.hero__media {
  position: absolute;
  inset: -1px;                   /* overscan by 1px to eliminate lines */
  z-index: 0;
}
.hero-video {
  position: absolute;
  inset: 0;
  width: 100%;
  height: 100%;
  display: block;                /* remove baseline gap from inline media */
  object-fit: cover;             /* no letterboxing */
  object-position: center;
  transform: translateZ(0);      /* compositor hint; avoids 1px seams */
  background: #000;              /* while buffering */
}

/* 4) Header must not draw a line when transparent */
.site-header {
  border: 0 !important;
  box-shadow: none !important;
  background: transparent;
}
.site-header.scrolled {
  background: rgba(0,0,0,.85);
  backdrop-filter: saturate(140%) blur(8px);
  border-bottom: 1px solid rgba(255,255,255,.08);
}

/* 5) Ensure hero/sections don’t add their own borders */
.hero, .hero__media, .hero__overlay {
  border: 0 !important;
}
Tiny JS sanity (keeps header transparent at top; dark on scroll)
document.addEventListener('DOMContentLoaded', () => {
  const header = document.querySelector('.site-header');
  const apply = () => window.scrollY > 8 ? header.classList.add('scrolled')
                                         : header.classList.remove('scrolled');
  apply();
  window.addEventListener('scroll', apply, { passive: true });
});
Why this works
100dvh accounts for the browser UI (URL bar), so no gap when it expands/collapses.
Overscanning inset:-1px hides sub-pixel edges that can show as 1-px lines on some GPUs.
display:block on the <video> removes the inline baseline gap.
Borders/shadows forced off while transparent so the header isn’t drawing a 1-px line.
Acceptance checks
No visible bars at the top or bottom at any zoom or viewport size.
Hero video fully covers; no letterboxing while playing or paused.
Header transparent at top, turns dark on scroll.
